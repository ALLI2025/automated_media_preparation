# Automated Media Preparation System (automated_media_preparation)

这是一个用于自动化培养基准备系统的机器视觉子项目，主要负责**液位实时检测**与**工业设备控制（如钧舵电爪）**。该系统通过 Python 后端与 LabVIEW 前端配合，实现高精度的工业灌装自动化。

## 🚀 核心功能

- **实时液位检测**：基于 OpenCV 的高性能视觉算法，支持动态 ROI 配置与满水线判定。
- **工业级稳定性**：内置高斯模糊、边缘锐化、众数统计及连续帧确认机制，有效过滤气泡、反光及水珠干扰。
- **多设备联动**：
    - **TCP 服务端**：提供标准 TCP 接口，支持 LabVIEW 等上位机实时获取检测结果。
    - **钧舵电爪控制**：集成驱动程序，实现对 ERG32-150 系列旋转电伺服电动夹爪的精准控制。
- **原子级数据交换**：推荐使用临时文件重命名技术，确保跨进程图像读取的完整性。

## 🛠️ 快速上手

### 1. 环境准备
确保已安装 Python 3.7+ 环境，并安装必要依赖：
```bash
pip install opencv-python numpy
```

### 2. 启动液位检测服务
```bash
cd surface_line
python DEMO.py
```
启动后，控制台将显示 `SERVER_READY`，此时可接受来自 LabVIEW 的 TCP 连接（默认端口：65432）。

### 3. 与 LabVIEW 联调
- **发送参数**：通过 TCP 发送格式如 `x1,y1,x2,y2,target_y` 的坐标字符串。
- **接收结果**：接收 `0x01` (已满) 或 `0x00` (不满) 字节信号。
- **状态复位**：发送 `RESET` 指令可清除当前检测参数。

## ⚙️ 算法优化方案 (针对工业环境)

为了保证 100% 的检测确定性，本项目在算法层面做了如下优化：
- **轻量预处理**：使用 3x3 高斯模糊，保留液面锐度。
- **连续帧校验**：只有连续 3 帧检测为满，才真正触发停泵信号。
- **直线支撑度过滤**：要求至少 15% 的检测宽度捕捉到清晰特征，防止水珠误导。
