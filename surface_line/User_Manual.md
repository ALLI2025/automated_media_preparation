# 自动化介质制备视觉反馈控制系统 V1.0 用户使用手册

## 1. 软件概述

### 1.1 软件简介
自动化介质制备视觉反馈控制系统 V1.0 是一款专为实验室自动化及工业精密灌装场景设计的高性能机器视觉中间件。本系统基于先进的计算机视觉技术与网络通信架构，旨在解决传统接触式液位传感器在无菌介质制备、微量分液及复杂容器灌装过程中存在的交叉污染、安装受限及精度不足等问题。

系统采用轻量级 C/S（客户端/服务器）架构，作为后台视觉服务运行。通过 TCP/IP 协议与上位机控制系统（如 LabVIEW、PLC 或其他自动化控制软件）无缝对接，实时接收控制指令，对采集到的流体图像进行高速处理与分析，并将液位状态（满/不满）以毫秒级低延迟反馈给主控系统，从而实现精准的闭环控制。

### 1.2 核心功能
本软件具备以下核心功能模块：

1.  **动态感兴趣区域（ROI）解析**：
    *   支持远程动态配置检测区域。用户无需修改代码或重启软件，只需通过 TCP 协议发送坐标参数（x1, y1, x2, y2），即可实时调整视觉检测窗口。
    *   适应性强，可灵活应对不同规格、不同位置的试管、培养皿或深孔板。

2.  **高精度液位识别算法**：
    *   内置优化的纵向梯度分析算法（Sobel算子改进版），专注于识别透明及半透明液体的弯月面特征。
    *   具备 `GRADIENT_THRESHOLD`（梯度阈值）自适应调节能力，有效过滤气泡、挂壁液滴及环境光反射造成的伪影干扰。

3.  **智能满水线判定**：
    *   支持自定义“满水线”目标位置（Target Y）。
    *   系统实时计算液面高度，并结合预设的像素容差（Tolerance），自动判定当前液位是否达标。
    *   输出二值化状态信号（0x01: 满 / 0x00: 不满），直接驱动电磁阀或蠕动泵停止工作。

4.  **鲁棒通信与文件处理**：
    *   采用非阻塞式（Non-blocking）Socket 通信机制，确保在网络波动或高负载下不阻塞主线程。
    *   引入独创的“鲁棒读取机制”（Robust Read），通过微秒级延时重试策略，完美解决了与图像采集端（如 LabVIEW 图像保存模块）之间的文件读写竞争（Race Condition），保证 7x24 小时连续运行不崩溃。

5.  **实时状态监控与日志溯源**：
    *   提供可视化的控制台“心跳”显示，实时展示当前的连接状态、图像读取状态及检测结果。
    *   自动记录全生命周期的运行日志（`server_debug.log`），涵盖启动、连接、异常报错及关键操作记录，便于故障排查与追溯。

### 1.3 性能指标
*   **处理速度**：单帧处理耗时 < 50ms（典型值），支持高达 20Hz 的实时检测频率。
*   **检测精度**：液位高度识别误差 < ±5 像素（取决于相机分辨率与安装距离）。
*   **通信延迟**：TCP 指令响应时间 < 10ms（本地环回模式）。
*   **稳定性**：支持连续运行 72 小时以上无内存泄漏，无意外退出。
*   **兼容性**：支持标准 BMP/JPG/PNG 图像格式输入，兼容各类工业相机及 USB 摄像头生成的图像流。

## 2. 运行环境

### 2.1 硬件配置要求
为确保软件稳定运行，建议计算机硬件满足以下最低配置：

*   **处理器 (CPU)**：Intel Core i5 6代及以上，或同级别 AMD 处理器（主频 2.0GHz 以上）。
*   **内存 (RAM)**：8GB DDR4 及以上（建议 16GB 以获得更佳的多任务处理性能）。
*   **硬盘**：至少 256GB SSD（固态硬盘），建议预留 10GB 以上空间用于临时图像缓存及日志存储。
*   **网络接口**：千兆以太网口（Gigabit Ethernet）或本地环回适配器（Localhost）。
*   **显示器**：分辨率 1920x1080 及以上（用于调试及监控查看）。
*   **图像采集设备**：支持 GenICam 标准的工业相机（如大恒图像、海康机器人等）或标准 USB 摄像头。

### 2.2 软件环境要求
*   **操作系统**：Windows 10 / 11 专业版或企业版（64位）。
*   **运行依赖库**：
    *   Python 3.8 或更高版本。
    *   OpenCV-Python (4.5+)：用于图像处理核心算法。
    *   NumPy (1.19+)：用于矩阵运算及数据分析。
*   **网络环境**：需开放 TCP 端口 65432（默认端口），确保防火墙未拦截该端口的入站连接。

## 3. 使用方法

### 3.1 软件安装与部署

1.  **获取软件包**：
    从提供的光盘或指定服务器下载 `Automated_Media_Prep_Vis_V1.0.zip` 压缩包。

2.  **解压部署**：
    将压缩包解压至 D 盘或 E 盘根目录下的英文路径文件夹中（例如：`D:\NOBELab\automated_media_preparation\surface_line`）。
    *注意：请避免使用包含中文字符或过长深度的路径，以免引发未知的文件读取错误。*

3.  **环境检查**：
    *   确保系统已安装 Python 环境。
    *   在安装目录下打开命令提示符（CMD），输入 `python --version` 检查版本。
    *   若未安装依赖库，请运行 `pip install opencv-python numpy` 进行安装。

### 3.2 程序运行

1.  **启动服务**：
    *   进入软件安装目录。
    *   双击运行 `DEMO.py`（或使用批处理脚本 `Start_Server.bat`）。
    *   此时将弹出一个黑色背景的控制台窗口。

2.  **启动状态确认**：
    *   观察控制台输出，当出现以下信息时，表示软件已启动成功并进入监听状态：
        ```text
        === 液面检测服务 ===
        [TCP服务器] 正在监听 127.0.0.1:65432...
        [TCP服务器] 等待 LabVIEW 连接并发送 ROI+满水线...
        SERVER_READY
        ```
    *   *注意：LabVIEW 等上位机可以通过扫描控制台输出的 `SERVER_READY` 或直接尝试 TCP 连接来确认服务已就绪。*

### 3.3 仪器连接与系统联调

本软件作为服务端（Server），需要配合上位机（Client）使用。以下以 LabVIEW 为例说明连接过程：

1.  **建立 TCP 连接**：
    *   在 LabVIEW 中使用“TCP Open Connection”函数。
    *   **Address**: `localhost` 或 `127.0.0.1`（若在同一台电脑运行）。
    *   **Port**: `65432`。
    *   连接成功后，控制台将显示：`New connection from ('127.0.0.1', xxxxx)`。

2.  **图像流配置**：
    *   上位机需控制相机采集图像，并将待检测的图像实时保存为本地文件。
    *   **保存路径**：必须为软件目录下的 `temp\temp.bmp`。
    *   *说明：软件会自动循环读取该路径下的图片进行分析。*

3.  **发送检测参数**：
    *   上位机需向本软件发送包含 ROI 和满水线信息的参数数组。
    *   **数据格式**：JSON 数组字符串 `[x1, y1, x2, y2, target_y]`。
        *   `x1, y1`: ROI 矩形左上角坐标。
        *   `x2, y2`: ROI 矩形右下角坐标。
        *   `target_y`: 目标满水线在整幅图像中的绝对 Y 坐标。
    *   **示例**：发送字符串 `[506, 234, 764, 490, 383]`。
    *   发送成功后，控制台将显示：`✅ Updated params: (506, 234, 764, 490, 383)`。

4.  **接收反馈结果**：
    *   软件会以约 20Hz 的频率向 LabVIEW 发送状态字节。
    *   上位机需循环调用“TCP Read”函数读取数据。
    *   **返回值定义**：
        *   `0x00` (HEX: 00)：表示液面**未达到**设定高度（继续加液）。
        *   `0x01` (HEX: 01)：表示液面**已达到**或超过设定高度（停止加液）。

### 3.4 运行状态监控与调试

在软件运行过程中，用户可以通过控制台输出的“心跳日志”实时监控系统健康状况。

**日志格式示例：**
`[120] 📷OK | Detect: ⚪ | Sent: b'\x00'`

**字段解析：**
*   `[120]`: 当前循环计数，每 20 次计数打印一次（约 1 秒）。
*   `📷OK`: **图像读取状态**。
    *   `OK` 表示成功读取到 `temp.bmp`。
    *   `FAIL` 表示文件不存在或被占用（软件会自动重试）。
*   `Detect: ⚪/🟢`: **视觉检测结果**。
    *   `⚪` (白色圆点)：液面未满。
    *   `🟢` (绿色圆点)：液面已满。
*   `Sent: b'\x00'/b'\x01'`: **网络发送数据**。指示实际发给上位机的指令字节。

### 3.5 常见问题与故障排除

1.  **控制台显示 `Bind failed: Address already in use`**
    *   **原因**：端口 65432 被占用，通常是软件重复开启导致的。
    *   **解决**：关闭所有已打开的黑色控制台窗口，重新运行一次程序即可。

2.  **控制台显示 `📷FAIL` 且无法变回 `OK`**
    *   **原因**：上位机未开始保存图片，或保存路径错误，或者文件读写冲突。
    *   **解决**：
        *   检查 LabVIEW 程序，确保其正在向 `temp` 文件夹下写入 `temp.bmp`。
        *   **进阶优化（建议）**：LabVIEW 保存图片时，建议先存为 `temp_tmp.bmp`，然后再调用系统命令将其重命名（Move/Replace）为 `temp.bmp`。这种“原子替换”操作可以彻底避免 Python 读取时的冲突。

3.  **液位检测不准（明明满了却显示不满）**
    *   **原因**：光照变化导致梯度特征不明显，或 ROI 范围设置不当。
    *   **解决**：
        *   调整光源亮度，使液面弯月面清晰可见。
        *   检查发送的 `x1, y1, x2, y2` 坐标是否准确框住了试管区域。
        *   微调代码中的 `GRADIENT_THRESHOLD` 参数（需联系开发人员）。

4.  **连接断开**
    *   **现象**：控制台显示 `Connection closed` 或 `Socket error`。
    *   **解决**：通常是上位机停止运行导致的。本软件会自动进入待机模式，重启上位机并重新连接即可恢复，无需重启本软件。

### 3.6 连续灌装与瓶子切换逻辑 (重要)

为确保工业生产中的确定性，本系统内置了**多重稳定性机制**：

1.  **连续帧确认 (Temporal Filtering)**：
    *   系统默认配置为 `CONFIRM_FRAMES = 3`。
    *   **原理**：只有当视觉系统连续 3 帧（约 150ms）都检测到液面到达目标线时，才会发送 `0x01`。这能有效过滤掉因水滴飞溅、瞬间反光导致的单帧误报。
    *   **状态观察**：控制台若显示 `🟡 (1/3)` 或 `🟡 (2/3)`，表示正在确认中，只有变为 `🟢 (3/3)` 才会触发停泵。

2.  **抗干扰算法**：
    *   采用了**高斯模糊**和**中值统计**算法，能够自动忽略瓶身上的细小反光点和水垢干扰。
    *   要求至少 30% 的检测宽度都捕捉到清晰液面特征，否则视为无效帧。

3.  **工业环境建议**：
    *   **背光照明**：建议在瓶子背面放置白色柔光板（背光），使液面弯月面形成清晰的黑线，这是保证 100% 准确率的最佳实践。
    *   **原子写入**：LabVIEW 保存图片时建议使用“临时文件名 + 重命名”的方式，避免文件读写冲突。

## 4. 个性化说明与注意事项

1.  **图像坐标系说明**：本软件遵循计算机视觉标准坐标系，原点 (0,0) 位于图像**左上角**，X 轴向右增加，Y 轴向下增加。因此，“液位越高”意味着其 Y 坐标数值**越小**。设置 `target_y` 时请务必注意此规律。
2.  **日志清理**：软件长期运行会产生 `server_debug.log` 文件。虽然文件增长缓慢，但建议定期（如每季度）手动备份或清理该文件，以释放磁盘空间。
3.  **非侵入式设计**：本软件运行时仅读取图片文件，不占用相机硬件驱动，因此可以与相机厂商自带的预览软件（如 GalaxyView）同时运行，方便调试光路。
